API - Description
FireApp Service

To use these REST-API interfaces you have to authenticate by sending a token with the key "token" in the header of your request.
If the request returns null it means that the token is invalid.
All request and responses are sent via the HTTP protocol and all objects are transmitted in the Json format.
The standard method for all requests is GET if not described else.

You can find a description of all domain objects and DTOs here: object description

{parameter} : this text should be replaced by the client using the API.
Type : this is a data type defined in the domain of the API.
"text" : name of a variable or a value.

FireEvents

URL RETURN VALUE DESCRIPTION
http://ip:port/events/..
 ../upload bool Returns "true" if the FireEvent was inserted or updated.
Transmit the FireEvent via POST in the body of the request.
 ../checkid/{sourceId}/{eventId} bool Returns "true" if the ID is not already used by another FireEvent.
"SourceId", Int32 : the sourceId of the FireEvent.
"EventId", Int32 : the eventId of the FireEvent.
 ../all FireEvent[] Returns an array of all FireEvents.
To use a filter on these FireEvents send you can send following headers. You can also use several filters at once:
 Return all FireEvents with a timestamp between start- and enddate:
  startDate : {yyyy-MM-dd}.
  endDate : {yyyy-MM-dd}.
 Return all FireEvents of the given EventTypes:
  events : EventTypes starting with capital letters seperated with ',' (e.g. "Alarm,Prealarm").
  events : "All" --> returns all EventTypes.
 Return all FireEvents of the given sourceId:
  sourceId : {id} (e.g. "2").
  sourceId : "-1" --> returns FireEvents from all sourceIds.
 ../active FireEvent[] Returns an array of all active FireEvents.
To use a filter on these FireEvents send you can send following headers. You can also use several filters at once:
 Return all FireEvents with a timestamp between start- and enddate:
  startDate : {yyyy-MM-dd}.
  endDate : {yyyy-MM-dd}.
 Return all FireEvents of the given EventTypes:
  events : EventTypes starting with capital letters seperated with ',' (e.g. "Alarm,Prealarm").
  events : "All" --> returns all EventTypes.
 Return all FireEvents of the given sourceId:
  sourceId : {id} (e.g. "2").
  sourceId : "-1" --> returns FireEvents from all sourceIds.
 ../getcsv HttpResponseMessage Returns all FireEvents as a CSV file.
 ../source/{sourceId} FireEvent[] Returns an array of all FireEvents from a distinct FireAlarmSystem.
"SourceId", Int32 : the id of the FireAlarmSystem.
 ../id/{sourceId}/{eventId} FireEvent[] Returns a distinct FireEvent.
"SourceId", Int32 : the sourceId of the FireEvent.
"EventId", Int32 : the eventId of the FireEvent.
 ../target/{sourceId}/{targetId} FireEvent[] Returns an array of all FireEvents of a distinct target.
"SourceId", Int32 : the sourceId of the FireEvents.
"TargetId", String : the targetId of the FireEvents.
 ../typeyear/{eventType}/{year} Int32[12] Returns an array with the number of FireEvents that occured in the given. Each column represents one month.
Only counts FireEvents of the given "EventType".
"EventType", Enum : the "EventType" of the FireEvents you want to count.
"Year", Int32 : the year in which the FireEvents occured.
 ../unregistered FireEvent[] Returns an array of FireEvents with no matching FireAlarmSystem.
Only returns the latest FireEvent per sourceId.

FireAlarmSystems

URL RETURN VALUE DESCRIPTION
http://ip:port/fas/..
 ../upload bool Returns "true" if the FireAlarmSystem was inserted or updated.
Transmit the FireAlarmSystem via POST in the body of the request.
 ../uploadbulk int Returns the number of upserted FireAlarmSystems or a negative error code
 "-1" : invalid or no token.
 "-2" : the User is not an admin.
 "-3" : an error occured.
Transmit the array of FireAlarmSystems via POST in the body of the request.
 ../checkid/{id} int Checks if an id is already used by another FireAlarmSystem.
If not, the sent id is returned, else a new unused id is returned.
"Id", Int32 : the id you want to check.
 ../all FireAlarmSystem[] Returns an array of all FireAlarmSystems.
 ../active FireAlarmSystem[] Returns an array of all FireAlarmSystems where there is an active FireEvent with a matching "sourceId".
 ../getcsv HttpResponseMessage Returns all FireAlarmSystems as a CSV file.
 ../uploadcsv HttpResponseMessage Upsert FireAlarmSystems from a CSV file.
Convert the CSV file into a byte array that you write into a String seperated by single whitespaces and send it via POST in the body of the request.
e.g. "105 100 59 110 97 109 101 13 10 50 59 70 70 32 ..."
 ../unregistered int[] Returns the "sourceId"s of the FireEvents where there is no FireAlarmSystem with an "id" matching the "sourceId" of the FireEvent.
 ../id/{id} FireAlarmSystem[] Returns the FireAlarmSystem with a matching "id".
"Id", Int32 : the "id" of the FireAlarmSystem.
 ../members/{id}/{type} object[] Returns an array of Usergroups of the given type of a FireAlarmSystem.
"Id", Int32 : the "id" of the FireAlarmSystem.
"Type", String : the type of object you want.
 "fb" for FireBrigades.
 "sg" for ServiceGroups.
 everything else returns all Usergroups.
 ../Users/{id}/{type} User[] Returns an array of Users of the given type from a FireAlarmSystem.
"Id", Int32 : the "id" of the FireAlarmSystem.
"Type", String : the type of Users you want.
 "fb" for Users with the "userType" FireFighters.
 "sg" for Users with the "userType" Servicemembers.
 "fas" for Users with the "userType" FireSafetyEngineers.
 everything else returns all types of Users.
 ../changemember/{fas}/{type}/{member}/{operation} Int32 Returns an error code.
 "1" : the operation was successful.
 "0" : the User is not logged in.
 "-1" : the User is not an admin.
 "-2" : a parameter was null or invalid.
 "-3" : an error occured.
"Fas", Int32 : the "id" of the FireAlarmSystem.
"Type", String : the type of member you want to change.
 "fb" for FireBrigades.
 "sg" for ServiceGroups.
"Member", Int32 : the "id" of the member you want to change.
"Operation", String : the type of operation you want to perform.
 "add" to add the member.
 "delete" to delete the member.

Users

URL RETURN VALUE DESCRIPTION
http://ip:port/User/..
 ../upload bool Returns "true" if the User was inserted or updated.
Transmit the User via POST in the body of the request.
 ../uploadbulk int Returns the number of upserted Users or a negative error code
 "-1" : invalid or no token.
 "-2" : the User is not an admin.
 "-3" : an error occured.
Transmit the Users as array via POST in the body of the request.
 ../delete/{username} bool Returns "true" if the User was deleted.
"username", String : the "id" of the User you want to delete.
 ../all User[] Returns an array of all Users.
 ../uploadcsv HttpResponseMessage Upsert Users from a CSV file.
Convert the CSV file into a byte array that you write into a String seperated by single whitespaces and send it via POST in the body of the request.
e.g. "105 100 59 110 97 109 101 13 10 50 59 70 70 32 ..."
 ../getcsv HttpResponseMessage Returns all Users as a CSV file.
 ../checkid/{id} bool Returns "true" if the id is not already used by another User.
"Id", String : the id you want to check.
 ../checkpassword Int32 Returns an error code.
 "1" : the password is safe.
 "0" : the password does not fit the criteria.
 "-1" : the password is too easy.
 "-2" : an error occured.
Transmit the User via Post in the body of the request.
 ../authenticate string Returns new token or "null" if the login credentials were incorrect.
Transmit the UserLogin via POST in the body of the request.
 ../getUser User Returns the User that is associated with the token if the token is valid or "null" if it is invalid.
 ../Usertype User[] Returns all Users with a "userType" that matches with one of the UserTypes.
Transmit the array of UserTypes via POST in the body of the request.
 ../id/{username} User[] Returns a distinct User.
"Username", String : the "id" of the User.
 ../active User[] Returns an array of all Users with a valid token.
 ../inactive User[] Returns an array of all Users with an invalid token.
 ../authobjects/{username}/{type} object[] Returns an array of objects that are associated with this User.
"Username", String : the "id" of the User.
"Type", String : the type of authorized object you want.
 "groups" returns the FireBrigades, ServiceGroups or FireAlarmSystems that are associated with this User.
 "fas" returns all FireAlarmSystems that are directly or indirectly associated with this User.
 ../changeauthobject/{username}/{authobj}/{operation} Int32 Adds or removes an authorized object of a User.
Returns an error code.
 "1" : the operation was successful.
 "0" : the User is not logged in.
 "-1" : the User is not an admin.
 "-2" : the User can not have authorized objects.
 "-3" : one of the parameters was null.
 "-4" : the operation does not exist.
 "-5" : an error occured.
"Username", String : the "id" of the User.
"Authobj", Int32 : the id of the authorized object.
"Operation", String : the type of operation you want to perform.
 "add" adds the id to the authorized objects of the User.
 "delete" deletes the id from the authorized objects of the User.
 ../resetpassword/{username} bool Generates a new password for the User with an "id" matching "username" and sends the new login credentials to the User's email account.
Returns "true" if the password was reset.
"username", String : the "id" of the User you want to reset the password of.
 ../changepassword bool Changes the password of the logged in User to the password that was transmitted.
Returns "true" if the password was changed.
Transmit the password via POST in the body of the request.

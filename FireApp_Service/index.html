API - Interface Description
FireApp Service

To use these REST-API interfaces you have to authenticate by sending a token with the key "token" in the header of your request.
If the request returns null it means that the token is invalid.

URL RETURN VALUE DESCRIPTION
http://ip:port/events/..
 ../update bool true if update was successful
transmit FireEvent via POST in Body
 ../checkid/{sourceId}/{eventId} bool true if id does not already exist
 ../all FireEvent[] Array of all FireEvents
 ../source/{sourceId} FireEvent[] returns all FireEvent from one FireAlarmSystem
 ../id/{sourceId}/{eventId} FireEvent[] returns a specific FireEvent
 ../target/{sourceId}/{targetId} FireEvent[] returns all FireEvents of a distinct target of a FireAlarmSystem
 ../type/{sourceId}/{eventType} FireEvent[] returns all FireEvents from a FireAlarmSystem with the given EventType
 ../type/{eventType} FireEvent[] returns all FireEvents with the given EventType
 ../time/{sourceId}/{startTime}/{endTime} FireEvent[] returns a list of all FireEvents with a matching sourceId and and a Timestamp between startTime and endTime
 ../time/{startTime}/{endTime} FireEvent[] returns a list of all FireEvents with a Timestamp between startTime and endTime
 ../date/{sourceId}/{year}/{month}/{day} FireEvent[] returns a list of all FireEvents of a FireAlarmSystem on the given date
 ../date/{sourceId}/{year}/{month} FireEvent[] returns a list of all FireEvents of a FireAlarmSystem in the given year and month
 ../date/{sourceId}/{year} FireEvent[] returns a list of all FireEvents of a FireAlarmSystem in the given year
 ../typeyear/{eventType}/{year} Int32[12] returns an array with the number of FireEvents of the given EventType that occured in the given year.
Each column represents one month
URL RETURN VALUE DESCRIPTION
http://ip:port/active/..
 ../all FireEvent[] returns a list of all active FireEvents
 ../type/{eventType} FireEvent[] returns a list of all active FireEvents of the given EventType
 ../source/{sourceId} FireEvent[] a list of active FireEvents with a matching sourceId
 ../id/{sourceId}/{targetId} FireEvent[] a list of active FireEvents with a matching sourceId and targetId
 ../type/{sourceId}/{eventType} FireEvent[] returns a list of all active FireEvents with a matching sourceId an of the given EventType
 ../date/{sourceId}/{year}/{month}/{day} FireEvent[] returns all active FireEvents from the given sourceId at the given date
 ../date/{sourceId}/{year}/{month} FireEvent[] returns all active FireEvents from the given sourceId in the given month and year
 ../date/{sourceId}/{year} FireEvent[] returns all active FireEvents from the given sourceId in the given year
URL RETURN VALUE DESCRIPTION
http://ip:port/fas/..
 ../upload bool returns true if the insert was successful
transmit FireAlarmSystem via POST in Body
 ../checkid/{id} int returns parameter id if it is not already used by another FireAlarmSystem or new unused id
 ../all FireAlarmSystem[] returns a list of all FireAlarmSystems
 ../active FireAlarmSystem[] returns a list of all FireAlarmSystems with active FireEvents
 ../id/{id} FireAlarmSystem[] returns a FireAlarmSystem with a matching id
 ../addfirebrigade/{id}/{firebrigade} bool returns true if the FireBrigade was added
 ../addservicemember/{id}/{servicemember} bool returns true if the ServiceGroup was added
URL RETURN VALUE DESCRIPTION
http://ip:port/fb/..
 ../upload bool returns true if the insert was successful
transmit FireBrigade via POST in Body
 ../delete/{id} bool returns true if FireBrigade was deleted
 ../checkid/{id} int returns parameter id if it is not already used by another FireAlarmSystem or new unused id
 ../all FireBrigade[] returns a list of all FireBrigades
 ../id/{id} FireBrigade[] returns a FireBrigade with a matching id
URL RETURN VALUE DESCRIPTION
http://ip:port/service/..
 ../upload bool returns true if the insert was successful
transmit ServiceGroup via POST in Body
 ../delete/{id} bool returns true if ServiceGroup was deleted
 ../checkid/{id} int returns parameter id if it is not already used by another FireAlarmSystem or new unused id
 ../all ServiceGroup[] returns a list of all ServiceGroups
 ../id/{id} ServiceGroup[] returns a ServiceGroup with a matching id
URL RETURN VALUE DESCRIPTION
http://ip:port/user/..
 ../upload bool returns true if the insert was successful
transmit UserLogin via POST in Body
 ../delete/{username} bool returns true if User was deleted
 ../all User[] returns a list of all Users
 ../checkid/{id} bool true if id does not already exist
 ../authenticate string returns new token or null if login data was incorrect.
transmit UserLogin via POST in Body
 ../getuser User returns User if token is valid or null if it is invalid
 ../id/{username} User[] returns a distinct user
 ../active User[] returns a list of all Users that have created a token in the last 365 days
 ../inactive User[] returns a list of all Users that have not created a token in the last 365 days

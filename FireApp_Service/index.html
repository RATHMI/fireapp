API - Interface Description
FireApp Service

To use these REST-API interfaces you have to authenticate by sending a token with the key "token" in the header of your request.
If the request returns null it means that the token is invalid.

FireEvents

URL RETURN VALUE DESCRIPTION
http://ip:port/events/..
 ../upload bool Returns true if the FireBrigades was inserted not updated.
Transmit the FireEvent via POST in the body of the request.
 ../checkid/{sourceId}/{eventId} bool Returns true if the id is not already used by another FireEvent.
SourceId : the sourceId of the FireEvent.
EventId : the eventId of the FireEvent.
 ../all FireEvent[] Returns an array of all FireEvents.
 ../active FireEvent[] Returns an array of all active FireEvents.
 ../getcsv HttpResponseMessage Returns all FireEvents as a CSV file.
 ../source/{sourceId} FireEvent[] Returns an array of all FireEvent from a distinct FireAlarmSystem.
SourceId : the Id of the FireAlarmSystem.
 ../id/{sourceId}/{eventId} FireEvent[] Returns a distinct FireEvent.
SourceId : the sourceId of the FireEvent.
EventId : the eventId of the FireEvent.
 ../target/{sourceId}/{targetId} FireEvent[] Returns an array of all FireEvents of a distinct target.
SourceId : the sourceId of the FireAlarmSystem.
TargetId : the targetId of the FireEvent.
 ../typeyear/{eventType}/{year} Int32[12] Returns an array with the number of FireEvents that occured in the given year where each column represents one month.
Only counts FireEvents of the given EventType.
EventType : the EventType of the FireEvents you want to count.
Year : the year in which the FireEvents occured.

FireAlarmSystems

URL RETURN VALUE DESCRIPTION
http://ip:port/fas/..
 ../upload bool Returns true if the FireBrigades was inserted not updated.
Transmit the FireAlarmSystem via POST in the body of the request.
 ../uploadbulk int Returns the number of upserted FireAlarmSystems or a negative error code
Transmit the array of FireAlarmSystems via POST in the body of the request.
 ../checkid/{id} int Checks if an id is already used by another FireAlarmSystem.
If not the sent id is returned else a new unused id is returned.
Id : the id you want to check.
 ../all FireAlarmSystem[] Returns an array of all FireAlarmSystems.
 ../active FireAlarmSystem[] Returns an array of all FireAlarmSystems where there is an active FireEvent with the same sourceId.
 ../getcsv HttpResponseMessage Returns all FireAlarmSystems as a CSV file.
 ../uploadcsv HttpResponseMessage Upsert FireAlarmSystems from a CSV file.
Transmit the CSV file as byte array via POST in the body of the request.
 ../unregistered int[] Returns the sourceIds of the FireEvents where there is no FireAlarmSystem with an id matching the sourceId.
 ../id/{id} FireAlarmSystem[] Returns the FireAlarmSystem with a matching id.
Id : the id of the FireAlarmSystem.
 ../members/{id}/{type} object[] Returns an array of the FireBrigades and or ServiceGroups of a FireAlarmSystem.
Id : the id of the FireAlarmSystem.
Type : the type of object you want.
 "fb" for FireBrigades.
 "sg" for ServiceGroups.
 everything else returns both.
 ../users/{id}/{type} User[] Returns an array of Users of the given type from a FireAlarmSystem.
Id : the id of the FireAlarmSystem.
Type : the type of User you want.
 "fb" for Users from FireBrigades.
 "sg" for Users from ServiceGroups.
 "fas" for Users from the FireAlarmSystem itself.
 everything else returns all types of Users.
 ../changemember/{fas}/{type}/{member}/{operation} Int32 Returns an error code.
 1 : the operation was successful.
 0 : the User is not logged in.
 -1 : the User is not an admin.
 -2 : a parameter was null or invalid.
 -3 : an error occured.
Fas : the id of the FireAlarmSystem.
Type : the type of member you want to change.
 "fb" for FireBrigades.
 "sg" for ServiceGroups.
Member : the id of the member you want to change.
Operation : the type of operation you want to perform.
 "add" to add the member.
 "delete" to delete the member.

 ../uploadbulk int Returns the number of upserted FireBrigades or a negative error code
Transmit the FireBrigades as array via POST in the body of the request.
 ../delete/{id} bool Returns true if the FireBrigade was deleted.
Id : the id of the FireBrigade you want to delete.
 ../checkid/{id} int Checks if an id is already used by another FireBrigade.
If not the sent id is returned else a new unused id is returned.
Id : the id you want to check.
 ../all FireBrigade[] Returns an array of all FireBrigades.
 ../getcsv HttpResponseMessage Returns all FireAlarmSystems as a CSV file.
 ../uploadcsv HttpResponseMessage Upsert FireBrigades from a CSV file.
Transmit the CSV file as byte array via POST in the body of the request.
 ../id/{id} FireBrigade[] Returns the FireBrigade with a matching id.
Id : the id of the FireBrigade.
 ../users/{id} User[] Returns all Users that are associated with this FireBrigade.
Id : the id of the FireBrigade.
 ../fas/{id} FireAlarmSystem[] Returns all FireAlarmSystems that are associated with this FireBrigade.
Id : the id of the FireBrigade.

Users

URL RETURN VALUE DESCRIPTION
http://ip:port/user/..
 ../upload bool Returns true if the User was inserted not updated.
Transmit the User via POST in the body of the request.
 ../uploadbulk int Returns the number of upserted Users or a negative error code
Transmit the Users as array via POST in the body of the request.
 ../delete/{username} bool Returns true if the User was deleted.
Id : the id of the User you want to delete.
 ../all User[] Returns an array of all Users.
 ../uploadcsv HttpResponseMessage Upsert Users from a CSV file.
Transmit the CSV file as byte array via POST in the body of the request.
 ../getcsv HttpResponseMessage Returns all Users as a CSV file.
 ../checkid/{id} bool Returns true if the id is not already used by another User.
Id : the id you want to check.
 ../checkpassword Int32 Returns an error code.
 1 : the password is safe.
 0 : the password does not fit the criteria.
 1 : the password is too easy.
Transmit the User object via Post in the body of the request.
 ../authenticate string Returns new token or null if login data was incorrect.
Transmit the UserLogin via POST in the body of the request.
 ../getuser User Returns the User if the token is valid or null if it is invalid.
 ../usertype User[] Returns all Users with a UserType that matches with one of the UserTypes.
Transmit the array of UserTypes via POST in the body of the request.
 ../id/{username} User[] Returns a distinct User.
Id : the id of the user.
 ../active User[] Returns an array of all Users with a valid token.
 ../inactive User[] Returns an array of all Users with an invalid token.
 ../authobjects/{username}/{type} object[] Returns an array of objects that are associated with this User.
Username : the username of the User.
Type : the type of authorized object you want.
 "groups" returns the FireBrigades, ServiceGroups or FireAlarmSystems that are associated with this User.
 "fas" returns all FireAlarmSystems that are associated with this User.
 ../changeauthobject/{username}/{authobj}/{operation} Int32 Adds or removes an authorized object of a User.
Returns an error code.
 1 : the operation was successful.
 0 : the User is not logged in.
 -1 : the User is not an admin.
 -2 : the User can not have authorized objects.
 -3 : one of the parameters was null.
 -4 : the operation does not exist.
 -5 : an error occured.
Username : the username of the User.
Authobj : the id of the authorized object.
Operation : the type of operation you want to perform.
 "add" adds the id to the authorized objects of the User.
 "delete" deletes the id from the authorized objects of the User.
 ../resetpassword/{username} bool Generates a new password for the User with a matching username and sends the new login credentials via to the User's email account.
Returns true if the password was reset.
Username: the username of the User you want to reset the password of.
 ../changepassword bool Changes the password of the logged in User to the password that was transmitted.
Returns true if the password was changed.
Transmit the password via POST in the body of the request.

Email

URL RETURN VALUE DESCRIPTION
http://ip:port/email/..
 ../help bool Returns an exit code as integer.
 1 : email was sent.
 0 : User is not logged in
 -1 : an error occured
Transmit the text of the email as string via POST in the body of the request.
